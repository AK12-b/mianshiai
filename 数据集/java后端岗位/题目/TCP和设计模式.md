一、TCP 协议
1.问题：说说 TCP 的三次握手？
答案：核心目的：同步双方初始序列号，确保双向通信。流程：1) 客户端发 SYN (seq=x)；2) 服务器回 SYN+ACK (seq=y,ack=x+1)；3) 客户端发 ACK (ack=y+1)。状态变化：客户端 SYN_SENT→ESTABLISHED，服务器 LISTEN→SYN_RCVD→ESTABLISHED。
2.问题：说说 TCP 的四次挥手？
答案：核心：可靠关闭全双工通道。流程：1) 客户端发 FIN；2) 服务端回 ACK（半关闭）；3) 服务端发 FIN；4) 客户端回 ACK，进入 TIME_WAIT (2MSL)。TIME_WAIT 防止 ACK 丢失，避免老报文干扰新连接。
3.问题：TCP 超时重传机制是为了解决什么问题？
答案：解决网络丢包问题。逻辑：发送方启动 RTO 定时器，超时未收 ACK 则重传，采用指数退避策略。RTO 随 RTT 动态调整，连续 15 次重传失败则断开连接，确保可靠传输。
二、TCP/IP 模型
1.问题：TCP/IP 四层模型是什么？
答案：1) 链路层：物理传输（以太网 / Wi-Fi）；2) 网络层：IP 寻址路由（路由器）；3) 传输层：TCP/UDP 端到端通信；4) 应用层：HTTP/FTP 等协议，面向用户应用。
三、设计模式
1.问题：请描述简单工厂模式的工作原理？
答案：集中对象创建：工厂类含静态方法，按输入类型（如 json/xml）返回对应实例。客户端依赖抽象接口，无需手动 new。适用于产品固定场景，违背开闭原则。
2.问题：什么是原型模式？一般用在什么场景？
答案：通过拷贝已有对象创建新对象，实现 Cloneable 并重写 clone ()。适用：创建成本高、配置微调、结构复杂差异小场景。注意浅拷贝需手动深拷贝引用类型。
3.问题：什么是模板方法模式？一般用在什么场景？
答案：父类固定算法骨架，可变步骤抽象，子类实现。适用：多子类流程相同、细节不同（如支付流程）。框架常用，如 Spring JdbcTemplate 封装固定逻辑。
4.问题：什么是策略模式？一般用在什么场景？
答案：算法封装为独立类（同一接口），运行时动态选择。解决 if-else 冗余，适用于行为分支多、需动态切换算法（如促销活动、报表导出）场景。
5.问题：请用一句话概括，什么是设计模式？为什么要用？
答案：前人总结的软件设计经典解决方案，复用思路、解耦提效、优化架构。分创建型（工厂）、结构型（代理）、行为型（观察者），主流框架广泛应用。
